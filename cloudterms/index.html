1. Cloud Computing: On-demand delivery of IT resources over the internet with pay-as-you-go pricing.
2. IaaS (Infrastructure as a Service): Cloud service providing virtualized computing resources like servers, storage, and networks.
3. PaaS (Platform as a Service): Cloud platform providing development tools, databases, and middleware for building apps.
4. SaaS (Software as a Service): Ready-to-use software applications delivered over the internet.
5. Public Cloud: Cloud infrastructure offered over the internet by providers like AWS, Azure, GCP.
6. Private Cloud: Cloud resources dedicated to a single organization, either on-premises or hosted.
7. Hybrid Cloud: Combination of public and private clouds working together.
8. Multi-Cloud: Using cloud services from multiple providers to avoid vendor lock-in.
9. Virtualization: Creating virtual versions of hardware, OS, or storage to maximize efficiency.
10. Containerization: Packaging apps and dependencies into lightweight, portable containers (e.g., Docker).
11. Serverless Computing: Running apps without managing servers; resources scale automatically (e.g., AWS Lambda).
12. Edge Computing: Processing data near the source (edge devices) instead of centralized cloud data centers.
13. Cloud Storage: Storing data online instead of local drives, accessible anytime, anywhere.
14. Cloud Security: Practices and tools to secure data, apps, and infrastructure in the cloud.
15. Elasticity: The ability of cloud resources to scale up or down automatically based on demand.
16. Scalability: Expanding resources to handle growth in workloads.
17. Pay-as-you-go: Billing model where users pay only for resources used.
18. Cloud Migration: Moving apps, data, and workloads from on-premises to the cloud.
19. Cloud Native: Apps built specifically to run in cloud environments, often using microservices.
20. Microservices: Breaking apps into smaller, independent services that communicate via APIs.
21. API (Application Programming Interface): A set of rules enabling apps to communicate with each other.
22. Load Balancer: Distributes incoming traffic across multiple servers for performance and reliability.
23. CDN (Content Delivery Network): Globally distributed servers that deliver content faster by caching close to users.
24. IAM (Identity and Access Management): Service to control access to resources securely.
25. Encryption: Converting data into unreadable code to protect it during storage or transmission.
26. Availability Zone (AZ): Physically separate data center within a cloud provider‚Äôs region.
27. Cloud Region: Geographic area with multiple data centers (AZs).
28. High Availability: Designing systems to minimize downtime and ensure reliability.
29. Disaster Recovery: Cloud strategies to restore services after unexpected failures.
30. Backup-as-a-Service (BaaS): Cloud-based solution for data backup and recovery.
31. DevOps: Collaboration between development and operations teams to automate software delivery.
32. Continuous Integration (CI): Regularly merging code into a shared repo with automated builds/tests.
33. Continuous Deployment (CD): Automatically releasing code to production after passing tests.
34. Orchestration: Automating management of multiple containers and services (e.g., Kubernetes).
35. SLA (Service Level Agreement): Formal contract defining performance guarantees by a cloud provider.
36. Cloud Cost Optimization: Managing resources efficiently to reduce cloud expenses.
37. Cloud Compliance: Adhering to laws, regulations, and policies in cloud usage.
38. Cloud Monitoring: Tools and processes to track performance, uptime, and security.
39. Observability: Advanced monitoring of logs, metrics, and traces to understand system health.
40. Cloud Native Security: Integrating security into cloud-native apps and infrastructure.
41. Zero Trust: Security model requiring continuous verification, never assuming trust.
42. Multi-tenancy: Multiple customers sharing the same cloud infrastructure while keeping data separate.
43. Fault Tolerance: System‚Äôs ability to continue working even if part of it fails.
44. Auto Scaling: Automatic adjustment of resources based on usage demand.
45. Kubernetes: Open-source system for automating containerized app deployment and management.
46. Docker: Platform for developing, shipping, and running apps inside containers.
47. API Gateway: Service that manages and secures APIs at scale.
48. Cloud Marketplace: Online store offering third-party apps and services for the cloud.
49. Cloud Bursting: Expanding workloads into the public cloud when private resources run out.
50. Vendor Lock-In: Difficulty in moving workloads between cloud providers due to proprietary tools.
